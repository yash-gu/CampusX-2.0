# 🚀 CampusX AWS Deployment Guide

## 📋 Prerequisites

1. **AWS Account** with appropriate permissions
2. **AWS CLI** installed and configured
3. **Docker** installed and running
4. **Domain name** (optional, for custom URL)

## 🛠️ AWS Services Used

- **ECS (Elastic Container Service)** - Container orchestration
- **ECR (Elastic Container Registry)** - Docker image storage
- **RDS** - MongoDB database
- **Application Load Balancer** - Traffic distribution
- **IAM Roles** - Security permissions
- **Secrets Manager** - Environment variables
- **CloudWatch Logs** - Logging

## 🔧 Setup Instructions

### 1. Configure AWS CLI
```bash
aws configure
# Enter your AWS Access Key ID
# Enter your AWS Secret Access Key
# Enter default region (us-east-1)
# Enter default output format (json)
```

### 2. Create Environment File
```bash
cp .env.example .env
# Edit .env with your actual values
```

### 3. Create IAM Roles

Create the following IAM roles:

#### ECS Task Execution Role
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "ecr:GetAuthorizationToken",
        "ecr:BatchCheckLayerAvailability",
        "ecr:GetDownloadUrlForLayer",
        "ecr:BatchGetImage",
        "logs:CreateLogGroup",
        "logs:CreateLogStream",
        "logs:PutLogEvents"
      ],
      "Resource": "*"
    }
  ]
}
```

#### ECS Task Role
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "secretsmanager:GetSecretValue"
      ],
      "Resource": "arn:aws:secretsmanager:*:*:secret:campusx/*"
    }
  ]
}
```

### 4. Set up MongoDB on AWS RDS

1. Go to AWS RDS console
2. Create a new MongoDB instance
3. Set up security group to allow ECS access
4. Note the connection string

### 5. Store Secrets in AWS Secrets Manager

Create these secrets:
- `campusx/mongodb-uri` - Your MongoDB connection string
- `campusx/jwt-secret` - Your JWT secret key

### 6. Create ECS Cluster

```bash
aws ecs create-cluster --cluster-name campusx
```

### 7. Create Security Groups

#### Backend Security Group
- Port 5001 inbound from frontend security group
- Port 5001 inbound from load balancer

#### Frontend Security Group
- Port 80 inbound from load balancer
- Port 80 inbound from backend security group

### 8. Create Application Load Balancer

1. Go to EC2 console → Load Balancers
2. Create Application Load Balancer
3. Set up target groups for frontend (port 80) and backend (port 5001)
4. Configure health checks

### 9. Deploy the Application

```bash
# Make deploy script executable
chmod +x deploy.sh

# Run deployment
./deploy.sh
```

## 🔍 Manual Deployment Steps

If the automated script doesn't work, follow these steps:

### 1. Build and Push Docker Images

```bash
# Build images
docker build -t campusx-backend:latest ./backend
docker build -t campusx-frontend:latest ./frontend

# Tag for ECR
docker tag campusx-backend:latest <aws-account-id>.dkr.ecr.<region>.amazonaws.com/campusx-backend:latest
docker tag campusx-frontend:latest <aws-account-id>.dkr.ecr.<region>.amazonaws.com/campusx-frontend:latest

# Push to ECR
docker push <aws-account-id>.dkr.ecr.<region>.amazonaws.com/campusx-backend:latest
docker push <aws-account-id>.dkr.ecr.<region>.amazonaws.com/campusx-frontend:latest
```

### 2. Create ECS Task Definition

Use the `task-definition.json` file generated by the deploy script or create manually in the ECS console.

### 3. Create ECS Service

```bash
aws ecs create-service \
  --cluster campusx \
  --service-name campusx-service \
  --task-definition campusx \
  --desired-count 1 \
  --launch-type FARGATE \
  --network-configuration "awsvpcConfiguration={subnets=[subnet-12345],securityGroups=[sg-12345],assignPublicIp=ENABLED}" \
  --load-balancers "targetGroupArn=arn:aws:elasticloadbalancing:<region>:<account-id>:targetgroup/campusx-frontend,containerName=frontend,containerPort=80"
```

## 🌐 Domain Setup (Optional)

### 1. Route 53 Configuration

1. Go to Route 53 console
2. Create a hosted zone if you don't have one
3. Create an A record pointing to your Load Balancer

### 2. SSL Certificate

1. Request a free SSL certificate through AWS Certificate Manager
2. Attach to your Load Balancer listener

## 📊 Monitoring and Logging

### CloudWatch Logs
- Logs are automatically sent to `/ecs/campusx` log group
- Set up CloudWatch Alarms for monitoring

### Health Checks
- Load Balancer health checks for both services
- ECS service health monitoring

## 🔒 Security Considerations

1. **IAM Roles**: Use least privilege principle
2. **Security Groups**: Restrict traffic to necessary ports only
3. **Secrets Manager**: Never store credentials in code
4. **VPC**: Consider running in a private subnet for production
5. **SSL**: Always use HTTPS in production

## 🚀 Production Optimizations

1. **Auto Scaling**: Configure ECS auto scaling
2. **CDN**: Use CloudFront for static assets
3. **Database**: Enable backups and read replicas
4. **Monitoring**: Set up CloudWatch dashboards
5. **Backup**: Regular database and file backups

## 🛠️ Troubleshooting

### Common Issues

1. **Container won't start**: Check CloudWatch logs
2. **Database connection**: Verify security group settings
3. **Load balancer issues**: Check target group health
4. **Permission errors**: Verify IAM roles

### Useful Commands

```bash
# Check ECS service status
aws ecs describe-services --cluster campusx --services campusx-service

# Check task status
aws ecs describe-tasks --cluster campusx --tasks <task-id>

# View CloudWatch logs
aws logs tail /ecs/campusx --follow

# Update service
aws ecs update-service --cluster campusx --service campusx-service --force-new-deployment
```

## 💰 Cost Optimization

1. **Use FARGATE SPOT** for non-production workloads
2. **Right-size** your CPU and memory allocations
3. **Use Reserved Instances** for predictable workloads
4. **Monitor** CloudWatch costs and set up billing alerts
